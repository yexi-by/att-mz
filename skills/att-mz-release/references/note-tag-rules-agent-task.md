# Note 标签规则子代理任务契约

本契约用于把标准 `data/*.json` 的 `note` 标签中玩家可见文本整理成导入规则。执行者只按本文件、CLI 输出、工作区文件和任务说明工作，不读取项目源码、数据库或程序内部对象。

## 输入

- 只读文件：`<工作区>/note-tag-candidates.json`。
- 只读文件：`<工作区>/note-tag-rules.json`，作为当前草稿或已保存规则回填。
- 任务参数：`<发行版目录>`、`<工作区>`、`<游戏标题>`。

## 逻辑

- 按候选中的 data 文件名、标签名和 `sample_values` 判断是否为玩家可见文本。
- 选择玩家会在说明、任务、菜单、状态、地图提示、事件名牌或补充描述中看到的自然语言文本。
- 排除脚本、公式、资源名、ID、布尔/枚举、数值列表、资源引用、内部关联字段、开关名、坐标、文件路径和纯机器协议标签。
- 同一标签名在不同游戏里含义可能不同，必须以当前候选样例值为准。
- 必须交叉验证每个选中标签：至少用标签名、`sample_values`、data 文件分布、同标签多样本或已回填草稿中的两个证据互相核对；证据不足时排除并报告风险。
- 不确定时先看同一文件和同一标签的样例值分布；仍不确定时排除，并在完成报告写明风险。

## 输出

- 唯一可写文件：`<工作区>/note-tag-rules.json`。
- 顶层必须是对象，格式为 `{data文件名或文件模式: [note标签名, ...]}`。
- 只写候选中真实存在的精确标签名，不支持标签正则。
- 合法空结果是 `{}`，表示已确认没有可导入 Note 标签文本规则。

中性格式示例：

```json
{
  "<物品数据文件>.json": ["<玩家可见说明标签>"],
  "<地图文件模式>": ["<玩家可见名牌标签>"]
}
```

## 校验

写完后建议主代理运行：

```powershell
.\att-mz.exe --agent-mode validate-note-tag-rules --game <游戏标题> --input <工作区>/note-tag-rules.json --json
```

如果校验失败，只修 `<工作区>/note-tag-rules.json` 后重跑同一校验命令；不要绕过 CLI 导入，不要直接改游戏 `data/*.json` 的 `note` 字段。

## 完成报告

完成时报告：

- 改动文件。
- 检查过的 data 文件和标签数量。
- 选中的标签与排除的主要标签类别。
- 是否为空结果；为空时说明检查范围和空结果理由。
- 交叉验证摘要：选中标签依据、重点排除标签依据、空结果依据、仍需主代理确认的标签。
- 未解决风险。
- 建议主代理运行的校验命令。
