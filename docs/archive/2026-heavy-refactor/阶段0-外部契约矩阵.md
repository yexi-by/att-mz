# 阶段 0 外部契约矩阵

执行日期：2026-06-03

本文只记录阶段 0 护栏需要保护的外部可观察契约。它不是实现设计，也不作为后续阶段的新增需求来源。

## 契约矩阵

| 契约面 | 当前事实来源 | 外部承诺 | 阶段 0 护栏 | 后续重构风险 |
| --- | --- | --- | --- | --- |
| CLI 命令面 | `app/cli/parser.py`、`app/cli/dispatch.py`、`app/cli/commands/` | 子命令名、参数名、退出码和 stdout JSON 是 Agent 与用户脚本可见接口；默认不再需要 `--agent-mode` 或 `--json`。 | `tests/test_skill_protocol.py` 校验 Skill、README 和发行 README 中的入口命令能被 parser 识别；`tests/test_stage0_canaries.py` 通过公开 `main()` 跑主链路。 | 拆分命令适配层时不能让文档示例和 parser 分叉，也不能把进度写入 stdout。 |
| stdout JSON | `app/cli_main.py`、`app/cli/reports.py`、`app/agent_toolkit/reports.py` | 正常、业务错误、未知异常都输出统一 Agent JSON；长任务进度只写 stderr。 | 主链路金丝雀逐步解析每个 CLI stdout JSON，失败时以报告内容定位。 | 重构日志或进度时最容易污染 stdout，必须保持机器可解析。 |
| 工作区 JSON | `app/agent_toolkit/services/workspace.py`、`skills/*/references/workspace-schema.md` | `<工作区>/manifest.json` 是本轮导出文件和工作流阶段的事实来源；导出的规则 JSON 只描述本轮候选和可导入内容。 | `test_stage0_stale_workspace_optional_files_not_in_manifest_are_ignored` 固定旧 `plugin-source-rules.json`、`nonstandard-data-rules.json` 和旧 `nonstandard-data/` 不参与本轮 validate。 | 后续删除工作区旧逻辑时不能回到“文件存在即参与校验”的双事实来源。 |
| SQLite schema | `app/persistence/sql.py`、`app/persistence/*_records.py`、`tests/test_persistence.py` | 每个游戏独立数据库位于 `<ATT_MZ_HOME>/data/db/`；当前 schema 版本为 14；表名和字段承载译文、规则、术语、运行记录和文本索引状态。 | 阶段 0 不迁移 schema；主链路金丝雀覆盖新增游戏、导入规则、保存译文、写回和反馈反查。 | 统一 repository 或删除旧表时必须显式迁移或显式失败，不能静默兼容旧语义。 |
| Skill 协议 | `skills/att-mz/SKILL.md`、`skills/att-mz-release/SKILL.md`、各自 `references/` | 开发版默认 `uv run python main.py <命令> ...`；发行版默认 `.\\att-mz.exe <命令> ...`；发行版不得要求 Python、Rust、uv、maturin 或源码。 | `tests/test_skill_protocol.py` 固定 frontmatter、默认入口、references 文件集合和旧开关禁用。 | 修改 CLI 或发行版入口时必须同步两份 Skill，不能让 docs 覆盖 Skill。 |
| README 和发行说明 | `README.md`、`docs/release-readme.md` | 文档示例只描述当前可运行入口和用户可见行为，示例不得包含真实本机路径。 | `tests/test_skill_protocol.py` 只校验命令示例和旧开关边界，不绑定自然语言措辞。 | 后续文档改写不能让 README 成为 Agent 契约来源，也不能重新引入旧开关。 |
| 发行包布局 | `scripts/build_release.py`、`.github/workflows/release.yml` | 正式发行包只包含 exe、配置模板、README、许可证、字体、提示词、发行版 Skill、references 和空运行目录。 | `tests/test_release_package_layout.py` 直接调用 `copy_release_resources()` 与 `create_release_zip()`，固定发行版 Skill 映射、references、空目录和禁止清单。 | 本机不能代替 GitHub Actions 构建正式发行包；资源复制函数改变时必须保持发行包边界。 |
| prompt 隐私 | `prompts/`、`app/application/use_cases/translation_run.py`、`tests/test_translation_cache_context.py`、`tests/test_text_protocol.py` | 最终 user prompt 只包含模型完成翻译必需的可见输入、输出协议、质量要求和占位符保留约束；不得包含真实路径、数据库字段和内部定位。 | 阶段 0 未新增 prompt 组装逻辑；主链路金丝雀使用假模型验证 prompt 至少保留 `id`、`line_count` 和占位符协议。 | 后续合并翻译上下文时必须继续测试最终 user prompt 文本，不把 `location_path`、`translated_text` 或真实路径暴露给模型。 |
| native contract | `app/native_contract.py`、`app/native_quality.py`、`app/native_write_plan.py`、`app/native_javascript_ast.py`、`tests/test_native_adapters.py` | Python 侧通过 JSON payload 调用 Rust 扩展；`native_contract_version()` 必须满足当前契约版本；线程数受 Rust 侧和 `ATT_MZ_RUST_THREADS` 影响。 | 阶段 0 保留既有 native adapter 测试；性能基线记录 Rust 线程数。 | 后续把扫描和写回迁移到 Rust 时必须同步契约版本和 adapter 测试，不能让 Python/Rust payload 字段漂移。 |

## 扫描预算观测点

阶段 0 只记录理论最大扫描对象，暂不设绝对秒数阈值。

| 命令或阶段 | `GameData` 加载 | 候选扫描 | `TextScope` 构建 | AST 扫描 | quality gate | write plan |
| --- | --- | --- | --- | --- | --- | --- |
| `prepare-agent-workspace` | 1 次翻译源视图 | 插件配置、事件指令、Note、占位符、术语候选各 1 轮 | 允许复用 warm index；冷路径最多 1 次 | 仅高风险或已有插件源码规则时 1 轮 | 不执行 | 不执行 |
| `validate-agent-workspace` | 1 次翻译源视图 | 本轮 manifest 中存在或数据库已启动的规则支线才扫描 | 1 次当前文本范围 | 仅本轮 manifest 或数据库要求时 1 轮 | 不执行写回级 gate | 不执行 |
| `translate` | 1 次翻译源视图 | 前置硬闸按需扫描外部规则和占位符 | 1 次当前文本范围，warm index 可用时应复用 | 仅插件源码支线需要时扫描 | 每批译文保存前执行 | 不执行 |
| `quality-report` | 1 次翻译源视图 | 外部规则、占位符和残留规则检查 | 1 次当前文本范围 | 仅插件源码支线需要时扫描 | Rust quality counts/details | 只在 `--include-write-probe` 时执行 |
| `write-back` | 1 次翻译源视图和 1 次当前运行写入目标 | 外部规则、占位符和残留规则检查 | 1 次当前文本范围 | 写回计划按插件源码运行映射需要扫描 | 写回前必须通过 | Rust write plan 1 次 |
| `verify-feedback-text` | 1 次翻译源视图和 1 次当前运行视图 | 不重新导入规则，只反查反馈原文 | 1 次当前文本范围 | 不扫描插件源码运行硬编码 | 不执行 | 不执行 |

## 阶段 0 新护栏

- `tests/test_stage0_canaries.py`：公开 CLI 主链路金丝雀；stale workspace 金丝雀。
- `tests/test_skill_protocol.py`：Skill、README、发行 README 与 parser 协议护栏。
- `tests/test_release_package_layout.py`：发行资源复制和 ZIP 空目录护栏。

这些护栏只固定外部行为，不固定旧结构函数名或内部 helper 调用次数。
