# 阶段 9 发布前审计

## Skill 同步结果

- 开发版 `skills/att-mz/SKILL.md` 和发行版 `skills/att-mz-release/SKILL.md` 的业务流程语义保持一致：识别游戏、注册、导出候选、导入规则、翻译、质量检查、写进游戏文件、试玩反馈修复。
- 允许差异只来自运行边界：开发版使用 `uv run python main.py <命令> ...`，可在用户明确要求源码排障时回到仓库修复；发行版使用 `.\att-mz.exe <命令> ...`，不得要求 Python、Rust、uv、maturin 或源码上下文。
- 两份 Skill 都要求 CLI stdout 输出 Agent JSON 报告，不再使用 `--agent-mode` 或 `--json`；命令失败时必须停止当前翻译流程并按当前入口重新校验。
- references 语义必须随发行包转换保持一致；发行脚本把发行版 Skill 转换为包内 `skills/att-mz/SKILL.md`，frontmatter 名称写成 `att-mz`。

## 发行方式

- 正式 Windows 发行包只由 GitHub Actions `release` 工作流生成，目标文件名为 `att-mz-windows-x86_64.zip`。
- `scripts/build_release.py` 默认只允许在 GitHub Actions 环境执行，发行包包含 exe、配置模板、README、许可证、字体、提示词、发行版 Skill、必要 references 和空的数据、日志、输出目录。
- 工作流发布前执行 `uv run basedpyright`、`uv run pytest`、`cargo fmt --manifest-path rust/Cargo.toml -- --check`、`cargo clippy --manifest-path rust/Cargo.toml --all-targets -- -D warnings`、`cargo test --manifest-path rust/Cargo.toml`、发行包构建和冒烟测试。
- GitHub Release 正文由 `scripts/extract_release_notes.py` 从 `CHANGELOG.md` 中当前 tag 段落提取；正文必须包含实际更新、验证命令和 `att-mz-windows-x86_64.zip` 下载信息。
