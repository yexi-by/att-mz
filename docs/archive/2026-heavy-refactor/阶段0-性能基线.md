# 阶段 0 性能基线

执行日期：2026-06-03

本文记录阶段 0 的可复查性能基线。阶段 0 不设置绝对秒数阈值，只记录现状、输入规模、Rust 线程数和重复扫描迹象。

## 样本和环境

| 项目 | 记录 |
| --- | --- |
| 样本来源 | 阶段 0 金丝雀同构的临时最小 RPG Maker MZ 游戏夹具 |
| 数据库准备 | 公开 CLI 完成 `add-game -> prepare-agent-workspace -> validate-agent-workspace -> 导入规则 -> translate -> quality-report -> write-back` |
| 模型 | 本地 OpenAI 兼容假模型；未调用真实模型 |
| 翻译规模 | 20 条正文，20 条成功，0 条 quality_error |
| 文件规模 | 42 个文件，17 个 `data/*.json`，2 个插件 JS，约 10.6 MB |
| benchmark 线程 | `ATT_MZ_RUST_THREADS=4` |
| 运行入口 | `uv run python scripts/<benchmark>.py ...` |
| 路径记录 | 实际运行使用系统临时工作目录；本文用 `<临时样本游戏目录>`、`<临时游戏数据库>` 占位 |

## benchmark 结果

### `benchmark_active_runtime_audit.py`

命令：

```powershell
uv run python scripts/benchmark_active_runtime_audit.py --sample <临时样本游戏目录> --game テストゲーム --db <临时游戏数据库> --runs 2 --rust-threads 4
```

结果：

| 指标 | 数值 |
| --- | --- |
| 状态 | ok |
| run_count | 2 |
| slowest_ms | 1220 |
| fastest_ms | 1216 |
| average_ms | 1218 |
| Rust 线程数 | 4 |
| 第 1 轮扫描文件 | 2 |
| 第 1 轮缓存 miss / rescan | 2 / 2 |
| 第 2 轮缓存 hit / rescan | 2 / 0 |
| 当前运行问题数 | 0 |

重复扫描迹象：第 1 轮建立缓存时重扫 2 个插件 JS；第 2 轮命中 2 个缓存且重扫 0 个文件，符合 warm cache 预期。

命令：

```powershell
uv run python scripts/benchmark_rebuild_active_runtime.py --sample <临时样本游戏目录> --game テストゲーム --db <临时游戏数据库> --runs 1 --rust-threads 4 --reset-active-data-from-origin
```

| 指标 | 数值 |

| elapsed_ms | 1374 |
| rust_plan_ms | 45 |
| file_replacement_ms | 47 |
| post_write_audit_ms | 0 |
| Rust 线程数 | 4 |
| 重置 `data_origin` 文件数 | 17 |
| planned_file_count / skipped_file_count | 16 / 3 |
| data_item_count / plugin_item_count | 20 / 0 |

重复扫描迹象：该样本未启动插件源码写回映射，AST 源扫描和运行扫描均为 0；写回计划与文件替换耗时都在单轮结果中可见。

### `benchmark_small_tasks.py`

命令：

```powershell
uv run python scripts/benchmark_small_tasks.py --sample <临时样本游戏目录> --game テストゲーム --db <临时游戏数据库> --runs 1 --rust-threads 4 --max-items 3 --manual-item-count 5
```

结果：

| 任务 | elapsed_ms | text_index_status | 关键内部耗时 | Rust 线程数 | 备注 |
| --- | ---: | --- | --- | ---: | --- |
| `rebuild-text-index` | 1268 | rebuilt | total 129ms；build_text_scope 4ms；write_text_index 12ms | 4 | indexed_count 20 |
| `quality-report` | 1202 | used | total 38ms；native_quality 27ms；build_index_scope 0ms | 4 | 0 个质量问题 |
| `translate --max-items 3` | 1263 | used | 无待翻译文本 | 0 | pending_count 0 |
| `import-manual-translations` | 1178 | used | 导入 5 条 | 0 | scope_mode `text_index` |
| `reset-translations --input` | 1176 | used | 重置 5 条 | 0 | 精确输入模式 |

重复扫描迹象：`quality-report`、`translate --max-items`、`import-manual-translations` 和 `reset-translations --input` 均报告使用 warm index；小样本下进程启动耗时明显大于内部阶段耗时，阶段 0 暂不据此设阈值。

## 扫描预算记录

| 对象 | 阶段 0 基线观察 |
| --- | --- |
| `GameData` 加载 | benchmark 未直接计数；由阶段 0 外部契约矩阵记录理论最大次数。 |
| 候选扫描 | 小任务热路径复用 text index；当前运行审计第 2 轮插件源码缓存 rescan 为 0。 |
| `TextScope` 构建 | `rebuild-text-index` 冷建 1 次；后续小任务报告 `text_index_status=used`。 |
| AST 扫描 | 当前运行审计扫描 2 个插件 JS；重建运行文件未触发插件源码 AST 写回扫描。 |
| quality gate | `quality-report` Rust native_quality 27ms；写回计划内 quality gate 由 `rebuild-active-runtime` 分段记录。 |
| write plan | `rebuild-active-runtime` Rust plan 45ms，planned_file_count 16。 |

剩余风险：样本是最小夹具，只能做快速基线；中大型真实游戏的重复全量扫描、插件源码 AST 和非标准 data 高风险支线仍需后续阶段用更大样本或 profiling 继续确认。
