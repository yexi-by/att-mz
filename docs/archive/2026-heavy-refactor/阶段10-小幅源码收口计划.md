## 风险与回滚

- 风险：当前工作树已有大量上一轮改动，阶段 10 实施时可能与未提交文件交叠。控制方式是只编辑文件清单中的文件，并在每个任务后查看 `git diff -- <文件>`，确认没有无关整理。
- 风险：`tests/scan_budget_contract.py` 作为测试侧模块可能触发导入路径问题。控制方式是使用现有项目已采用的 `from tests...` 导入风格，并由 `uv run basedpyright` 与 `uv run pytest tests/test_scan_budget.py` 验证。
- 风险：仓库外部私有脚本若导入 `app.scan_budget` 或 `app.json_path_protocol.jsonpath_to_location_path` 会失败。当前仓库证据显示它们不是生产契约；本轮不为仓库外未知调用保留兼容分支。
- 风险：删除 `app/plugin_text.__all__` 中的旧短名可能影响内部顶层导入。当前 `rg` 未发现测试或生产代码从 `app.plugin_text` 顶层导入该短名；如实施时发现新引用，应改为从 `app.plugin_text.common` 导入或改用明确名，不恢复中性模块旧别名。
- 回滚：若 Task 1 失败，只恢复 `app/application/use_cases/translation_run.py` 中删除的 `TranslationProgressState` 段落。
- 回滚：若 Task 2 失败，只恢复 `app/scan_budget.py` 并把 `tests/test_scan_budget.py` 导入改回 `app.scan_budget`；删除 `tests/scan_budget_contract.py`。
- 回滚：若 Task 3 失败，只恢复 `app/json_path_protocol/__init__.py` 和 `app/plugin_text/__init__.py` 的本轮删除内容；不要回退事件指令或插件文本其他阶段改动。
- 回滚命令只能面向阶段 10 触碰文件，例如 `git restore -- <文件>`；禁止使用 `git reset --hard` 或回退无关上一轮改动。
